# Scrutinize API Reference

This document explains how external applications can connect to Scrutinize.

## Connecting Locally Hosted Projects
If Scrutinize is running locally on port `8000` (via `docker compose up`), and your external frontend/backend is also running locally, use `http://localhost:8000` as the base URL. 

> [!TIP]
> **Docker to Docker:** If your external app is *also* running in a Docker container on the same machine, `localhost` will point to the external app's container itself, not Scrutinize. In this case, use `http://host.docker.internal:8000`.

## 1. Authentication

### Signup
Creates a new project space and returns API keys.

- **Endpoint**: `POST /v2/projects/signup`
- **Auth**: None
- **Request Body** (JSON):
```json
{
  "name": "my_project",
  "password": "my_secure_password"
}
```
- **Response** (201 Created):
```json
{
  "project_id": "uuid-string",
  "api_key": "scrutinize_sk_...",
  "client_key": "scrutinize_pk_..."
}
```

### Login
Retrieves API keys for an existing project.

- **Endpoint**: `POST /v2/projects/login`
- **Auth**: None
- **Request Body** (JSON):
```json
{
  "name": "my_project",
  "password": "my_secure_password"
}
```
- **Response** (200 OK):
```json
{
  "project_id": "uuid-string",
  "api_key": "scrutinize_sk_...",
  "client_key": "scrutinize_pk_..."
}
```

## 2. Ingestion (Upload)

Uploads media (Text, Audio, Video) to be chunked and vectorized.

- **Endpoint**: `POST /upload`
- **Auth Header**: `X-Project-Key: <your_api_key>`
- **Request Body** (Multipart Form-Data):
  - `file`: The binary file being uploaded.
- **Response** (200 OK):
```json
{
  "file_id": "uuid-string",
  "job_id": "uuid-string",
  "filename": "document.pdf",
  "modality": "text",
  "status": "pending",
  "message": "File accepted for processing"
}
```

## 3. Query (Chat / Search)

Ask the AI questions about your uploaded data.

- **Endpoint**: `POST /v2/search`
- **Auth Header**: `X-Client-Key: <your_client_key>` (or `X-Project-Key`)
- **Request Body** (JSON):
```json
{
  "query": "Summarize the latest document",
  "modality_filter": "all" // Optional: "text", "audio", "video", or "all"
}
```
- **Response** (200 OK):
```json
{
  "query": "Summarize the latest document",
  "answer": "Here is the summary generated by the LLM...",
  "sources": [
    {
      "file_id": "uuid",
      "filename": "document.pdf",
      "content": "Original text segment matched...",
      "score": 0.89
    }
  ]
}
```

## 4. Library Management

### View Library
List all files uploaded to your project.

- **Endpoint**: `GET /library`
- **Auth Header**: `X-Project-Key: <your_api_key>`
- **Response** (200 OK):
```json
[
  {
    "id": "uuid",
    "filename": "document.pdf",
    "modality": "text",
    "status": "indexed"
  }
]
```

### Delete File
Permanently delete a file and its vector data.

- **Endpoint**: `DELETE /library/{file_id}`
- **Auth Header**: `X-Project-Key: <your_api_key>`
- **Response** (200 OK):
```json
{
  "file_id": "uuid",
  "message": "File deleted successfully"
}
```
